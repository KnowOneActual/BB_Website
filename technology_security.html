Website Technology & Security Report

This page provides an overview of the technologies and practices that power my personal website, focusing on how I've enhanced its security, performance, and overall user experience.

The Technology Stack

My website is built with a modern and robust technology stack designed for performance, security, and scalability.

  • Frontend: HTML5, CSS3, and JavaScript.
  • Styling: Tailwind CSS (built from source via npm script).
  • Visuals: CSS Animations (for the animated gradient hero and noise texture).
  • Hosting & Serverless: Netlify (for seamless deployment, continuous integration, and serverless functions).
  • Serverless Functions (Node.js):
    • Weather Bot: Integrates with Google Gemini API and OpenWeatherMap API.
    • Contact Form: Utilizes Resend for secure email delivery and Cloudflare Turnstile for spam verification.

Key Improvements & Actions Taken:

1. Robust HTTP Security Headers & Policy

Action: Implemented a comprehensive `netlify.toml` file to define crucial HTTP security headers and established a `security.txt` file for a clear vulnerability reporting channel.

Why it matters: This provides a robust defense by:

  • Content-Security-Policy (CSP): Prevents Cross-Site Scripting (XSS) by whitelisting all allowed sources for scripts, styles, and images.
  • X-Frame-Options (DENY): Prevents clickjacking by disallowing the site from being embedded in iframes.
  • X-Content-Type-Options (nosniff): Mitigates MIME-sniffing attacks.
  • Permissions-Policy: Disables unwanted browser features (like geolocation, microphone) to enhance user privacy.

2. Code Refactoring for Security & Maintainability

Action: Moved all inline JavaScript from `index.html` into an external `script.js` file. Event handlers were refactored to be attached programmatically.

Why it matters: This was crucial for strengthening the Content Security Policy by removing the need for `'unsafe-inline'`, which significantly reduces the XSS attack surface. It also improves code organization and maintainability.

3. Hero Animation Performance Optimization

Action: Replaced the JavaScript-based Three.js hero animation with a lightweight, CSS-only animated gradient and noise texture.

Why it matters: This change removed a large JavaScript library, dramatically reducing the page's initial load size, speeding up render times, and improving the overall user experience without sacrificing a modern aesthetic.

4. Privacy Enhancement & Cleanup

Action: Removed the third-party Cloudflare Web Analytics script and disabled the service. A comprehensive `privacy.html` page was added to the site.

Why it matters: This move enhances user privacy by stopping all third-party analytics tracking. It also fixed several console errors related to script blocking and integrity mismatches. The new privacy policy provides clear transparency about the data the site *does* collect (e.g., via the contact form).

5. Accessibility Enhancements

Why it matters: These improvements ensure the site is more accessible to users with visual impairments and those using assistive technologies.
